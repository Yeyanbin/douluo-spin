## 各阶段详细任务

### 阶段 A：Handler 补全

```typescript
// A-1. applyCommon() 新增容貌变化
const appearanceChange = text.match(/容貌\s*([+-])\s*(\d+)/)
if (appearanceChange && !/ex级?无法提升/.test(text)) {
  const GRADES = ['F','E','D','C','B','A','S','EX']
  const idx = GRADES.indexOf(context.appearance.toUpperCase())
  if (idx >= 0) {
    const delta = Number(appearanceChange[2]) * (appearanceChange[1] === '+' ? 1 : -1)
    context.appearance = GRADES[Math.max(0, Math.min(GRADES.length - 1, idx + delta))]
  }
}

// A-2. switch 新增 case 'beastEncounter':
case 'beastEncounter': {
  if (!context.beast) break
  const gainMatch = text.match(/修为\s*\+\s*(\d+)/)
  const lossMatch = text.match(/修为掉落\s*(\d+)/) ?? text.match(/修为\s*-\s*(\d+)/)
  let delta = 0
  if (gainMatch) delta = Number(gainMatch[1])
  if (lossMatch) delta = -Number(lossMatch[1])
  if (delta !== 0) context.beast.cultivation = Math.max(10, context.beast.cultivation + delta)
  break
}
```

### 阶段 B：战斗数值化（约 180 个选项）

战斗结果分成六类，每类嵌入标准文本模式：

| 战斗结果 | 文本模式 | 影响 |
|----------|----------|------|
| 无伤碾压 | "无伤" + "斩杀/秒杀" | `等级+2，获得【XX】称号` |
| 轻伤获胜 | "轻伤战胜" | `等级+1` |
| 平手/无功过 | "平手""无功无过""僵持" | 无变化（保持现状） |
| 轻伤/无功过 | 明确说"轻伤""无功无过" | 无变化 |
| 重伤 | "重伤" 不含 "死亡""战死" | `等级-1` |
| 阵亡 | "战死" / "被斩杀" / "被击杀" | `lethal()` 已有 |

### 阶段 C：势力数值化（约 120 个选项）

每个势力的 5 阶段 × 3 路线选项嵌入模式：

| 阶段 | 嵌入模式 |
|------|----------|
| 启蒙期 | `获得【XX新秀】称号，等级+1` |
| 成长期 | `等级+1` |
| 担当期 | `等级+1` |
| 精英期 | `等级+1，获得【XX精英】称号` |
| 领袖期 | `等级+2，获得【XX领袖】称号` |
